Detailed Description

Implements the internal helper functions for emitting AArch64 machine code.

This file provides the concrete implementations for the low-level AArch64 instruction emitters declared in abi_arm64_emitters.h. Each function in this file is responsible for constructing a single, valid 32-bit AArch64 instruction word from its constituent parts (registers, immediates, etc.) and appending it to a code_buffer.

This module encapsulates all the bitwise logic for encoding ARM64 instructions, keeping the main abi_arm64.c file focused on the higher-level logic of argument classification and trampoline structure. This separation improves code readability and maintainability by isolating the complexities of the instruction set architecture.

Emits an AArch64 ADD (immediate) instruction.

Emits an ARM64 ADD(S) <Xd/Wd>, <Xn/Wn>, #imm instruction (add immediate to GPR).

Encodes ADD(S) <Wd|Xd>, <Wn|Xn>, #imm. Adds an immediate value to a register. This function automatically handles large immediates by generating a multi-instruction sequence if necessary.

  • Opcode (64-bit): 10_0_10001_... (0x91...)
  • Opcode (32-bit): 00_0_10001_... (0x11...)
Parameters
bufThe code buffer.
is64True for 64-bit operation (operands are X registers).
set_flagsTrue to update condition flags (emits ADDS).
destThe destination GPR.
baseThe source GPR.
immThe immediate value. Can be larger than 12 bits.

Emits an AArch64 LDR (immediate) instruction.

Emits an ARM64 LDR <Wt/Xt>, [Xn, #imm] instruction (load GPR from memory).

Encodes the LDR <Wt|Xt>, [<Xn|SP>, #pimm] instruction. This loads a 32-bit or 64-bit value from memory at the address base + offset into the destination register.

  • Opcode (64-bit): 11_111_00_1_01_... (base 0xB9400000)
  • Opcode (32-bit): 10_111_00_1_01_... (base 0x79400000)
Parameters
bufThe code buffer to write the instruction to.
is64True for a 64-bit load (LDR Xt), false for 32-bit (LDR Wt).
destThe destination general-purpose register (X0-X30 or W0-W30).
baseThe base general-purpose register (Xn or SP) for the memory address.
offsetThe 12-bit scaled immediate offset. Must be positive and a multiple of the access size (4 or 8).

Emits an AArch64 LDR <Qt>, [Xn, #imm] for a 128-bit load into a SIMD&FP register.

This instruction is used for loading 128-bit values, such as the long double type on Linux AArch64, from memory into a full 128-bit Q-register (which is the full width of a V-register).

  • Opcode: 00_111_10_1_01... (base 0x3DC00000)

baseThe base GPR for the memory address.

Emits instructions to load an arbitrary 64-bit immediate into a GPR.

Emits an instruction sequence (MOVZ/MOVK) to load an arbitrary 64-bit immediate into a GPR.

This is achieved by emitting a sequence of up to four "move wide" instructions: one MOVZ for the first 16-bit chunk (to zero the register) followed by up to three MOVK instructions for the remaining chunks. This is the standard AArch64 pattern for materializing a 64-bit constant.

Parameters
bufThe code buffer.
destThe destination GPR.
valueThe 64-bit immediate value to load.
